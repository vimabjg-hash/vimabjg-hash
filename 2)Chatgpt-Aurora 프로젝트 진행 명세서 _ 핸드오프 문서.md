
# Aurora 프로젝트 진행 명세서 / 핸드오프 문서

## 0. 프로젝트 개요
Aurora는 특정 사이트 전용이 아니라 "전체 사이트"에서 동작하는 브라우저 확장 프로그램이다.
핵심 기준은 사이트별 분기보다 "읽기 모드 / 쓰기 모드" 분기다.

즉,
- 읽기 모드에서는 읽기 기능이 정상 작동해야 한다.
- 쓰기 모드에서는 쓰기 기능이 정상 작동해야 한다.
- 특히 쓰기 모드에서는 입력/커서/포커스를 절대 방해하면 안 된다.

---

## 1. 절대 원칙
1. Aurora는 모든 사이트에서 읽기 모드 / 쓰기 모드 기능이 정상적으로 작동해야 한다.
2. 명세서는 더 잘게 쪼개서 하나씩 확인 및 점검하는 방식으로 진행한다.
3. 사이트 기준 차단보다 모드 기준 동작이 우선이다.
4. 특정 사이트만 되고 다른 사이트에서 안 되면 실패로 본다.
5. Claude Code 점검/수정 프롬프트에는 항상 아래 기준을 자동 포함해야 한다.
   - 읽기 모드 정의
   - 쓰기 모드 정의
   - 전체 사이트 동작 기준
   - 성공 기준
   - 예외 금지 조건
   - 실패 판정 기준

## 2. 읽기 모드 / 쓰기 모드 정의

### 읽기 모드
사용자가 "읽을 텍스트"를 보고 있거나 드래그한 상태.
예:
- 기사 본문
- 블로그 글
- 일반 웹페이지 본문
- 댓글 본문
- AI 답변 본문
- 코드블록을 포함한 읽기용 출력 영역

읽기 모드에서 기대 동작:
- 드래그 툴바 표시
- 번역
- 요약
- 질문
- 저장
- 팝업 결과 표시
- 패널에서 계속

### 쓰기 모드
사용자가 실제 입력 가능한 영역에서 글을 쓰거나 선택한 상태.
예:
- input
- textarea
- contenteditable
- role="textbox"
- 검색창
- ChatGPT / Gemini / Claude / Copilot 입력창
- 댓글 작성창
- 문서 에디터

쓰기 모드에서 기대 동작:
- 입력/커서/포커스 절대 방해 금지
- 다듬기
- 질문
- 바꾸기
- 복사
- 팝업에서 후속 질문
- 선택한 문장 기반 재작성

## 5. 현재 정상 동작하는 것

### 입력 안정성(P1) 해결 완료
다음 문제는 해결됨:
- Aurora 활성화 시 ChatGPT/Gemini 입력창 타이핑 막힘
- 커서가 사라짐
- Google 검색창 / 네이버 검색창 타이핑 문제

수정 요지:
- selection-toolbar.ts hide()에서 removeAllRanges() 제거
- selection-observer.ts onMouseUp에서 editable 내부 빈 선택 시 조기 return

현재 정상:
- ChatGPT 입력창 타이핑 정상
- Gemini 입력창 타이핑 정상
- Google 검색창 타이핑 정상
- 네이버 검색창 타이핑 정상
- 드래그 없이 클릭만 해도 커서 유지 정상

### 런타임 에러 해결 완료
문제:
- 번역 클릭 시 팝업 로딩중 + "text is not defined" 런타임 에러

원인:
- src/content/ui/floating-shell.ts constructor 내부에
  USER_PROMPTS[action as AiAction](text)
  참조가 남아 있었음

수정:
- USER_PROMPTS[action as AiAction](this.sourceText) 로 변경

현재 정상:
- 확장 프로그램 오류 사라짐
- 번역 정상 작동
- 결과 정상 출력

### RESULT 태그 제거 개선 완료
문제:
- [RESULT], [/RESULT] 태그가 팝업 결과에 간헐적으로 노출됨

수정:
- sanitizeResult(text) 추가
- showResult()에서 this.resultText = sanitizeResult(text) 사용
- resultEl.textContent도 sanitize 후 결과 사용

현재 상태:
- extractResult()가 놓쳐도 showResult() 진입 시점에 최종 제거되도록 보강됨
- 번역 / 요약 / 질문 / 다듬기 / 추가 질문 경로에 공통 적용

### AI 사이트 읽기/쓰기 기본 동작 상태
확인된 상태:
- AI 사이트 입력창 타이핑 정상
- AI 사이트 읽기 본문 드래그 툴바 대체로 정상
- Gemini 답변 본문 드래그는 읽기 모드 정상
- ChatGPT 입력창 드래그는 쓰기 모드 툴바 정상

## 7. 아직 남아 있거나 계속 점검해야 할 부분

### A. ChatGPT 특이 케이스
최근 상태:
- ChatGPT 답변 본문 드래그는 대체로 읽기 모드
- 하지만 코드블록 부분에서 쓰기 모드로 잘못 분류되던 문제가 있었음
- 이 부분은 보정 진행됨

계속 확인 필요:
- ChatGPT 답변 본문 전체(코드블록 포함)가 완전히 읽기 모드인지
- ChatGPT 입력창만 쓰기 모드인지

### B. 쓰기 모드 전체 사이트 안정화
목표:
- 일부 사이트만 아니라 전체 사이트에서 쓰기 모드 선택/팝업/바꾸기 흐름이 안정적이어야 함

특히 점검할 것:
- input
- textarea
- contenteditable
- role="textbox"
- 검색창
- 에디터 계열(ProseMirror, Slate, Lexical, Monaco, CodeMirror 등)

### C. 바꾸기(replace) 신뢰성
문제 축:
- 원문 영역을 찾을 수 없음
- source metadata 손실 가능성
- 일부 사이트는 직접 치환이 어려울 수 있음

원칙:
- replace 실패 시 버튼 라벨을 복사로 바꾸지 말 것
- 복사 버튼은 복사만 담당
- 실패 이유는 status 문구로 구체적으로 보여줄 것

### D. 드래그 후 우클릭
문제 축:
- 우클릭 시 선택 표시가 사라질 수 있음
- 최소한 Aurora 내부 selection snapshot은 유지되어야 함

아래를 새 채팅에 먼저 붙여넣고 시작하면 된다.

나는 Aurora 브라우저 확장 프로그램을 만들고 있다.

절대 원칙:
1. Aurora는 모든 사이트에서 읽기 모드 / 쓰기 모드 기능이 정상 작동해야 한다.
2. 명세서는 더 잘게 쪼개서 하나씩 확인 및 점검하는 방식으로 진행한다.
