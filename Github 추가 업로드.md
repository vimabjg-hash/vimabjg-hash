중학교 수준의 비유로 설명하자면, **"새로 만든 숙제를 가방에 넣고(add), 이름표를 붙여서(commit), 학교 사물함에 넣는(push) 과정"**과 같습니다.

---

### 🚀 추가 업로드(업데이트) 3단계 루틴

코드를 수정하거나 새로운 파일을 만든 후, 터미널(`C:\Aurora`)에서 아래 순서대로 입력하세요.

#### 1단계: 변경된 파일들 '장바구니'에 담기

내가 수정한 모든 내용을 업로드 후보로 올리는 작업입니다.

Bash

```
git add .
```

> **💡 팁:** `git status`라고 입력해 보면, 어떤 파일이 수정되었는지 빨간색/초록색으로 친절하게 보여줍니다.

#### 2단계: 업로드 내용에 '이름표' 붙이기

이번에 무엇을 수정했는지 짧은 메모를 남기는 단계입니다.

Bash

```
git commit -m "메인 화면 색상 변경 및 버그 수정"
```

> **⚠️ 주의:** 따옴표(`""`) 안에 내가 한 일을 적어주세요. 나중에 **Gemini Gems**가 이 메모를 보고 "아, 주인님이 이때 이런 수정을 하셨구나!"라고 파악하게 됩니다.

#### 3단계: 온라인 창고(GitHub)로 '발송'하기

실제로 인터넷에 올리는 마지막 단계입니다.

Bash

```
git push
```

> **✨ 결과:** 이제 비밀번호를 다시 물어보지 않고 바로 **촤르륵~** 올라갈 거예요!

---

### 🧐 만약 에러가 난다면? (딱 한 가지만 기억하세요)

혹시라도 `git push`를 했는데 에러가 난다면, 그건 **"누군가(혹은 내가 웹사이트에서 직접) 온라인 창고의 파일을 고쳐서, 내 컴퓨터와 순서가 꼬였다"**는 뜻입니다. 그럴 땐 당황하지 말고 아래 명령어를 먼저 치세요.

Bash

```
# 온라인에 있는 최신 내용을 내 컴퓨터로 먼저 가져오기
git pull origin main
```

그다음 다시 `add` -> `commit` -> `push` 순서로 진행하면 해결됩니다.
