# 1) 드래그 툴바 + 팝업 창을 합친 선택 기반 UX 시퀀스 최종본

## v2.0 — Persona + Tone & Manner 통합판

## 1-1. 한 줄 정의

```text
사용자가 텍스트를 선택하면
드래그 툴바는 "작업 + 페르소나"를 빠르게 고르게 하고,
팝업 창은 "결과 확인 / 즉시 삽입 / 패널 확장"을 담당한다.
```

---

## 1-2. 역할 분담 고정

### 드래그 툴바

```text
역할
- 기능 선택기
- 페르소나 빠른 선택기
- 즉시 닫기(X) 제공

금지
- 긴 결과 표시
- 복잡한 설정 노출
```

### 팝업 창

```text
역할
- 결과 확인
- 입력창 즉시 교체/삽입
- 페르소나/톤 재실행
- 패널 확장
```

---

## 1-3. 드래그 툴바 구조 최종 수정본

디렉터 요청 반영: **`X 닫기 아이콘` 추가**

### 1행: 핵심 액션

```text
번역 | 요약 | 다듬기 | 질문 | X
```

### 2행: 페르소나 프리셋

```text
🧹 미니멀 | 😈 비판 | 🎓 정의 | 💎 마스터
```

### 3행: 보조 제어

```text
더보기 | 고정 | 최근 실행
```

## X 닫기 아이콘 규칙

```text
- 현재 선택에 대한 드래그 툴바만 즉시 닫음
- 사이트/전역 비활성화는 아님
- 가장 빠른 일회성 종료
```

## 숨기기와의 관계

```text
기존 '숨기기'는 visible row에서 제거
→ X가 즉시 종료 역할 담당
→ 사이트별/전역 숨김은 더보기 또는 설정으로 이동
```

이 구조가 더 낫습니다.  
이유는 **`X`는 순간 종료**, **숨김은 설정 성격**이라 레벨이 다르기 때문입니다.

---

## 1-4. 페르소나 적용 규칙

업로드하신 MD의 4개 페르소나를 툴바/팝업에 아래처럼 녹입니다.

### 🧹 미니멀리스트

```text
역할
- 군더더기 제거
- 인칭/어조 보존
- 핵심만 정리
```

### 😈 악마의 대변인

```text
역할
- 논리 허점 3개 지적
- 비판적 관점 제공
- 반론/리스크 보기
```

### 🎓 사전적 정의

```text
역할
- 개념을 한 문장으로 쉽게 설명
- 초등학생도 이해 가능한 수준
```

### 💎 프롬프트 아키텍트

```text
역할
- 선택 텍스트/아이디어를 기반으로
- 상위 모델용 고급 프롬프트 템플릿 생성
```

---

## 1-5. 기능과 페르소나의 관계

### 기본 원칙

```text
기능 = 무엇을 할지
페르소나 = 어떤 방식으로 답할지
```

### 예시

```text
요약 + 🧹 미니멀
→ 핵심만 남긴 짧은 요약

질문 + 🎓 정의
→ 쉬운 설명 중심 답변

질문 + 😈 비판
→ 논리 허점/반론 중심 답변

다듬기 + 🧹 미니멀
→ 인칭 유지 + 군더더기 제거

질문/다듬기 + 💎 마스터
→ 상위 모델에 넣을 정교한 프롬프트 변환
```

---

## 1-6. 기본 자동 추천 규칙

속도를 위해, Aurora는 **기능 클릭 시 추천 페르소나를 자동 매칭**합니다.

```text
번역 → 🧹 미니멀 기본
요약 → 🧹 미니멀 기본
다듬기 → 🧹 미니멀 기본
질문 → 🎓 정의 기본
```

### 예외

```text
비판/검토 문맥 감지
→ 😈 비판 추천

"프롬프트", "Claude", "Gemini", "ChatGPT", "template" 문맥 감지
→ 💎 마스터 추천
```

### 사용자 제어

```text
- 사용자는 툴바 2행에서 직접 페르소나 변경 가능
- 클릭한 페르소나가 자동 추천보다 우선
```

---

## 1-7. 톤앤매너 실행 규칙

업로드하신 MD의 톤앤매너를 **UI 실행 레벨**로 번역하면 아래처럼 고정됩니다.

### Direct & Logical

```text
- 군더더기 없는 답변
- 인사치레 최소화
- 바로 결과부터 제시
```

### Identity Preserving

```text
- 나/내/너/네 관계 보존
- 저/제/당신으로 불필요하게 바꾸지 않음
- 입력창 교체 시 특히 강하게 적용
```

### Structured Output

```text
- 내부 처리 구조는 reasoning/conclusion 철학을 따름
- UI 출력은 사용자 친화형으로 단순화
- 팝업에서는 '핵심 결과' 우선
- 패널 확장 시 구조형 보기 허용
```

## 1-8. 선택 기반 시퀀스 최종 흐름

## Flow A. 읽기 영역

```text
텍스트 선택
→ 드래그 툴바 표시
→ 기능 선택
→ 필요 시 페르소나 선택/변경
→ 팝업 오픈(확인 모드)
→ 상태 표시
→ 결과 생성
→ 복사 / 다시 실행 / 패널에서 계속
```

## Flow B. 입력 영역

```text
입력창 텍스트 선택
→ 드래그 툴바 표시
→ 기능 선택
→ 필요 시 페르소나 선택/변경
→ 팝업 오픈(삽입 모드)
→ 상태 표시
→ 수정안/번역안 생성
→ 바로 교체 / 커서 위치 삽입 / 아래에 추가
```

## Flow C. 빠른 종료

```text
텍스트 선택
→ 툴바 표시
→ 필요 없으면 X 클릭
→ 현재 툴바 즉시 종료
```

## 1-9. 팝업 창에서의 페르소나/톤 노출 규칙

팝업은 툴바에서 고른 실행 컨텍스트를 **확인·변경·재실행**할 수 있어야 합니다.

### 상단 보조 상태

```text
작업명 | 현재 페르소나 | 상태 | 핀 | 패널로 보내기 | 닫기
```

### 팝업 내 보조 칩

```text
🧹 | 😈 | 🎓 | 💎
```

### 동작

```text
- 다른 페르소나 칩 클릭
→ 같은 원문으로 즉시 재실행

- 톤은 전역 기본값 유지
→ 필요 시 패널에서 상세 조정
```

즉, **툴바는 빠른 선택**, **팝업은 빠른 수정**입니다.

## 1-12. Identity Guard UX 반영

업로드 MD의 핵심 제약은 **인칭 보존**입니다.  
이건 UX에도 드러나야 합니다.

### 적용 규칙

```text
- 다듬기/번역 결과 상단에 '인칭 유지' 배지 선택 가능
- 기본 ON
- 사용자는 꺼도 되지만 기본은 고정 추천
```

### 오류 시 UX

```text
인칭 손상 감지
→ 팝업 상태를 '재교정 중'으로 표시
→ 사용자의 추가 클릭 없이 재실행
→ 수정 결과로 교체
```

### 이유

사용자 체감상

```text
"왜 내가 쓴 나/너가 저/제/당신으로 바뀌었지?"
```

이 불쾌감을 줄이는 것이 매우 중요하기 때문입니다.

## 1-13. 선택 기반 UX 최종 확정본

```text
[선택 기반 UX 시퀀스 최종본 v2.0]

1. 사용자가 텍스트를 선택한다
2. 드래그 툴바가 선택 영역 근처에 뜬다
3. 툴바 1행에서 기능(번역/요약/다듬기/질문)을 고른다
4. 툴바 2행에서 페르소나(🧹😈🎓💎)를 유지하거나 바꾼다
5. 필요 없으면 X로 현재 툴바를 즉시 닫는다
6. 기능 선택 즉시 팝업 창이 열린다
7. 팝업은 선택 위치에 따라 자동 모드를 결정한다
   - 읽기 영역: 확인 모드
   - 입력 영역: 삽입 모드
8. 팝업 상단에는 작업명 / 현재 페르소나 / 상태 / 핀 / 패널로 보내기 / 닫기가 있다
9. 팝업 본문에는 원문과 결과가 분리되어 표시된다
10. 결과 준비 후
   - 확인 모드: 복사 / 다시 실행 / 패널에서 계속
   - 삽입 모드: 바로 교체 / 커서 위치 삽입 / 아래에 추가 / 다시 실행
11. Identity Guard는 기본 활성화되어 인칭 보존을 우선한다
12. 짧은 작업은 팝업에서 끝내고, 긴 작업은 패널로 넘긴다
```

```text
- 글 다듬기
- 선택 구문 질문
```

## Journey C. 전역 기능에서 선택 기반으로 연결

```text
세로 사이드바 열기
→ 더보기 클릭
→ 미등록 기능 팝업 오픈
→ 기능 실행
→ 현재 선택 텍스트가 있으면 그 문맥으로 팝업 또는 패널 실행
```

### 의미

```text
전역 기능과 선택 문맥이 끊기지 않음
```

## Journey D. 짧은 작업에서 긴 작업으로 승격

```text
텍스트 선택
→ 툴바/팝업으로 1차 처리
→ 패널에서 계속 클릭
→ 우측 패널 오픈
→ 원문 / 결과 / 페르소나 / 톤 상태 그대로 전달
→ 긴 대화 / 비교 / 재작성 / 후속 질문
```

## 2-7. 패널의 역할 최종 고정

패널은 단순 결과창이 아닙니다.

```text
패널 = 지속형 작업 공간
```

### 패널이 가져야 할 것

```text
- 현재 작업 종류
- 원문 컨텍스트
- 현재 페르소나
- Identity Guard 상태
- 결과 이력
- 후속 대화
```

### 패널에서 좋은 기능

```text
- reasoning/conclusion 구조형 보기
- 페르소나 재실행
- 다른 기능으로 전환
- 최근 작업 비교
```

즉, **팝업은 짧게**, **패널은 깊게**입니다.

## 2-8. 전역 진입점과 선택 진입점의 충돌 방지 규칙

### 세로 사이드바

```text
전역 런처
```

### 플로팅 버튼

```text
전역 빠른 호출
```

### 드래그 툴바

```text
선택 문맥 즉시 실행
```

### 충돌 방지 원칙

```text
- 선택이 있으면 드래그 툴바 우선
- 선택이 없으면 플로팅 버튼/세로 사이드바 우선
- 같은 작업을 동시에 두 UI가 강하게 주장하지 않음
```

## 2-10. 전체 사용자 여정 최종 확정본

```text
[전체 사용자 여정 최종본 v2.0]

1. 사용자는 세로 사이드바, 플로팅 버튼, 드래그 툴바 중 하나로 Aurora에 진입한다
2. 세로 사이드바는 Aurora 런처이며, 펼침 상태에서 더보기를 통해 미등록 기능을 실행/등록/미등록/고정핀 설정할 수 있다
3. 플로팅 버튼은 패널/최근 작업을 여는 전역 손잡이이며, 드래그로 위치 이동 가능하다
4. 사용자가 텍스트를 선택하면 드래그 툴바가 우선 진입점이 된다
5. 드래그 툴바는 1행 기능, 2행 페르소나, 3행 보조 제어로 구성되고 X로 즉시 종료할 수 있다
6. 기능 선택 후 팝업 창이 열린다
7. 팝업은 읽기 영역이면 확인 모드, 입력 영역이면 삽입 모드로 자동 전환한다
8. 팝업은 결과 확인 / 즉시 삽입 / 패널 확장을 담당한다
9. 전역 기본 페르소나와 톤앤매너는 세로 사이드바 설정에서 관리하고, 선택 순간의 페르소나는 툴바/팝업에서 즉시 덮어쓸 수 있다
10. Identity Guard는 기본 활성화되며, 인칭 손상 감지 시 재교정 흐름으로 들어간다
11. 짧은 작업은 팝업에서 끝내고, 긴 작업은 패널에서 이어간다
12. 패널은 원문, 결과, 페르소나, 후속 대화를 유지하는 지속형 작업 공간이다
```
